VALIDACIÓN

Codigo de producto:
Validación de tipo de dato, para verificar que se ingrese un numero
Análisis de valor límite, para asegurarse que se un codigo valido entre 10000 y 99999
Validación de campo obligatorio, todo producto está asociado con un código

Nombre del producto:
Validación de campo obligatorio, para asegurarse de que el producto tenga un nombre que lo identifique


Fecha de compra:
Comprobación de formato, para asegurarse de que se ingrese una fecha


Precio de compra:
Validación de tipo de dato, para verificar que se ingrese un float
Comprobación de formato, para asegurarse de que luego de la coma solo haya dos dígitos


Código del proveedor:
Validación de tipo de dato, para verificar que se ingrese un numero
Verificacion de validez de datos, se rechaza la entrada si no es el numero que se corresponda con un codigo de proveedor registrado



CODIFICACIÓN:
Para la codificación de las prendas en la tienda de ropa, podríamos utilizar una combinación alfanumérica. Se podrían usar 3 letras y 4 números, las letras representarían el tipo de prenda (por ejemplo, CAM como camisa, PAN para pantalón, FAL para falda) y los números se asignarían secuencialmente a cada modelo especifico dentro de cada tipo de prenda (por ejemplo CAM0001 para cierto modelo de camisa, CAM0002 para otro modelo de camisa, etc.)
Para la codificación de los proveedores, también usaríamos una combinación alfanumérica con los números en forma secuencial. Podrían usarse 4 letras y 3 números, por ejemplo, MATE001 para un proveedor de materiales y TRAN001 para un proveedor de servicios de transporte.
